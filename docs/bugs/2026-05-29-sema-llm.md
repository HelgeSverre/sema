# sema-llm — Audit Findings (2026-05-29)

[2026-05-29 codebase audit](./2026-05-29-codebase-audit-index.md).
Line numbers as of commit `f609807` — re-confirm before fixing.

## P0 — Critical

### [LLM-1] API key in URL query string + user-controlled `model` in path
- **Location:** `crates/sema-llm/src/gemini.rs:36` and `:80`
- **Confidence:** 97% (re-verified against source)
- **Why:** Both `complete_async` and `stream_complete_async` build
  `format!("{}/models/{}:generateContent?key={}", base_url, model, api_key)`. On any reqwest
  failure, `LlmError::Http(e.to_string())` (line 48) embeds the full URL — **leaking the API key**
  into error values, logs, and any surface that renders the error. Every other provider sends the
  key as a header. Additionally `model` is request-controlled and interpolated into the path,
  enabling SSRF/path-injection (e.g. a model string of `../../v1/...`).
- **Fix:** Send the key via the `x-goog-api-key` header and build the URL without it;
  URL-encode/validate `model` before interpolation. (Also closes LLM-5.)

## P1 — High

### [LLM-2] `&result[..200]` byte-slice panics on multibyte tool result
- **Location:** `crates/sema-llm/src/builtins.rs:4238-4239`
- **Confidence:** 92%
- **Why:** Byte-based slice on a UTF-8 `String` in the `run_tool_loop` hot path; panics with
  "byte index 200 is not a char boundary" for any tool returning >200 bytes with a multi-byte
  char at the boundary. See Pattern B in the index.
- **Fix:** `result.chars().take(200).collect::<String>()` (or `floor_char_boundary(200)`).

### [LLM-3] `thread::sleep` inside `block_on` starves the tokio runtime
- **Location:** `crates/sema-llm/src/builtins.rs:3983-3984` and `4009-4010`
- **Confidence:** 90%
- **Why:** `do_complete_with_provider` / `do_complete_uncached` call
  `std::thread::sleep` during rate-limit retry. These run synchronously inside
  `runtime.block_on(...)` (e.g. `anthropic.rs:325`), blocking the runtime thread and preventing
  other tasks from progressing. `batch_complete` (concurrent `join_all`) can stall entirely on a
  single rate-limit hit.
- **Fix:** Use `tokio::time::sleep` and make the retry path async (or sleep before `block_on`).

### [LLM-4] User-controlled provider `base_url` → SSRF
- **Location:** `crates/sema-llm/src/openai.rs:647`
- **Confidence:** 88%
- **Why:** `llm/configure :base-url` passes a user string straight to `OpenAiProvider::named`
  (inherited by groq/xai/mistral/moonshot/`other` and `llm/configure-embeddings`). A Sema script
  or notebook can point the provider at an internal address (`http://169.254.169.254/...`,
  `http://localhost:9200/`) and the API key is sent there in the `Authorization` header.
- **Fix:** Reject `http://` and private/loopback IP ranges (or whitelist host suffixes) under the
  sandbox capability model.

## P2 — Moderate

### [LLM-5] API key leaked in `eprintln!` fallback log
- **Location:** `crates/sema-llm/src/builtins.rs:3948`
- **Confidence:** 85%
- **Why:** `eprintln!("Provider '{}' failed: {}, trying next...", provider_name, e)` prints the
  full error, which for Gemini includes the URL with `?key=` (see LLM-1) → key on stderr on every
  fallback.
- **Fix:** Log only error type/status. Fixing LLM-1 (header auth) also resolves this.

### [LLM-6] Cache key may be computed before model resolution
- **Location:** `crates/sema-llm/src/builtins.rs:3905-3937` (vs `compute_cache_key` at `:3836`)
- **Confidence:** 80%
- **Why:** When `request.model` is empty it is filled from the default; if the key is computed
  against the unresolved/empty model, interleaved `llm/with-cache` + `llm/set-default` across
  providers can collide on a cache key for different concrete models.
- **Fix:** Compute the cache key only after `request.model` is resolved to a concrete name.

### [LLM-7] `cosine_similarity` silently drops mismatched-dimension docs
- **Location:** `crates/sema-llm/src/vector_store.rs:148`
- **Confidence:** 80%
- **Why:** Returns `None` on dimension mismatch; `VectorStore::search` uses `filter_map`, so docs
  with a different embedding length are silently skipped (e.g. after switching embedding models a
  loaded store returns empty results with no diagnostic).
- **Fix:** Return an error from `search` on dimension mismatch (or count + surface skipped docs).

## Notes (not filed as findings)
- `track_usage` adds tokens before the over-limit check → "spent" overcounts by one call.
- `enforce_rate_limit` (builtins.rs ~4018) does `now - last` on `u64` ms timestamps; a backward
  clock adjustment underflows → effectively infinite sleep. Use `saturating_sub`/`wrapping_sub`.
- `sse.rs`/`ndjson.rs` use `from_utf8_lossy` per chunk; a multi-byte char split across a chunk
  boundary yields a replacement char + orphaned continuation bytes (silent corruption) for
  dense-Unicode providers.
